# Blue Archive Auto Script
![Python](https://img.shields.io/badge/-Python-000000?style=flat&logo=python)

**BAAS**は、GUIを備えたブルーアーカイブの自動化スクリプト（全サーバー対応）です。16:9の画面解像度（1280x720が最適）で実行するよう設計されており、ブルーアーカイブのすべてのタスクを管理し、完全な自動化を実現することを目的としています。

## 機能 Features
- **キャラクター好感度**: カフェテリアで1日8回タッチ、スケジュールで指定生徒を自動検索、毎日好感度を最大限上昇
- **メインストーリー**: 自動推し図（ノーマル4-28、ハード1-28、最新メインストーリー<28>対応済み）
- **カフェテリア**: 招待券で指定生徒を選択可能、カフェテリアのタッチ、カフェテリアの報酬
- **ショップ**: 指定された通常物品ショップとアリーナショップの自動購入と更新回数に対応
- **報酬獲得**: 毎日チーム体力、メール、アリーナ毎日報酬、総力戦累積スコア報酬、毎日タスク報酬、パスタスクポイントと報酬取得
- **体力クリア**: 任意のメインストーリーステージ（ノーマル/ハード）、特別委託、活動ステージを指定回数でクリア可能
- **スケジュール**: 指定キャラクターがいるスケジュールを優先実行、好感度が増加するスケジュールを優先、各エリアのスケジュール実行回数を指定
- **アリーナ**: アリーナチャレンジチケットがなくなるまでクリア、毎日報酬を自動獲得
- **製造**: シームレスな3段階製造、製造優先度と回数を指定、加速券の使用有無を指定
- **ももトーク**: 未完了の会話をすべて自動完了、ストーリーを完成させ、青光石を獲得
- **総力戦**: 総力戦チャレンジチケットをクリアして報酬を獲得（自動スコア調整機能は[このリポジトリ](https://github.com/pur1fying/BAAS_Cpp)で対応済み、**BAAS**に統合中）
- **戦術総合試験**: 試験開期間中に自動でチケットをクリア
- **ストーリー**: ワンクリックでメインストーリーストーリー、グループストーリー、ミニストーリーをクリア
- **活動**: ワンクリックで活動ストーリー、タスク、チャレンジ、格子歩きを実行（3サーバー活動は安定更新）

### 顕著な特徴：

- **16:9解像度のエミュレータはすべて実行可能で、1280x720に限定されません**
- **低スペックのパソコンで実行してもエラーが発生しません** プロセッサ速度が低い場合は、スクリーンショット速度を手動で調整して実行時間を延長できます
- **カスタムスケジュール（毎日固定リフレッシュ時間、実行間隔）**

## インストール Installation
  **インストールパスに中国語が含まれていないことを確認してください（QTフレームワークの制限）**
  ReleaseまたはQQグループからダウンロードパッケージを解凍し、`BlueArchiveAutoScript.exe`をダブルクリックして環境をインストールしてください。お待ちください。
  インストール完了後、BAASのUIインターフェースが自動的に起動します。同時に、pyinstallerで使用できるパッケージスクリプトも提供しています。自分でパッケージ化できます。詳細は`deploy/installer`フォルダに記載されています。
  [インストール関連ドキュメント](https://baas.wiki/usage_doc/install/choose_platform.html)

## バグ報告方法 How to Report Bugs
質問する前に、少なくとも5分間考えて準備してください。そうすれば、誰かがあなたを手伝うために5分間を費やすでしょう。

質問する前に、次のことを確認してください。
BAASの更新を確認し、最新バージョンを使用していることを確認します（プログラムを再起動すると自動的に最新バージョンに更新されます）。
予期しない動作の場合は、予期しない動作が発生したときのUIインターフェースのログ、エミュレータのスクリーンショット、またはビデオを提供してください。

## 開発
GitHubのissueと開発ドキュメントに要件を公開します。興味がある場合は開発に参加できます。**BAAS**へのプルリクエストを歓迎します。すべてのコード行を注意深く読みます。あ、そうだ。[開発ドキュメント](https://blog.lzwi.fun/blue_archive_auto_script/develop_doc/develop_guide)を読むことを忘れないでください。

1. **BAAS**の自動戦闘はYOLOv8を使用してゲーム内キャラクターを検出します。開発者の負担を軽減し、データ注釈に参加したい場合は、作者に連絡してください。
2. GUIを他の言語に適応させることを歓迎します（現在はEnglish、Simplified Chinese、日本語、한국말があります）
3. **BAAS**の宣伝を手伝いたい場合は、作者に連絡してください。宣伝方法は以下を含みますが、これらに限定されません：
    - BiliBiliで**BAAS**自動戦闘ライブを使用してスコアを調整するのを支援
    - さまざまなビデオプラットフォームで使用方法チュートリアルビデオを作成して宣伝
